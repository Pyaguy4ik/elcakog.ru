# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
#Разработка веб-ресурса для обучения китайскому языку
## 1. Основание для разработки
Разработка веб-приложения обусловлена необходимостью создания интерактивной платформы для изучения китайского языка, которая сочетает в себе функциональность словаря, систему тренировок и игровые элементы для повышения эффективности обучения и мотивации пользователей.
## 2. Назначение разработки
Создание программного комплекса для изучения китайских слов и предложений с возможностью добавления пользовательского контента, выбора слов для тренировки и использования различных игровых режимов для закрепления материала.
## 3. Цели разработки
•	Создание интерактивной системы тренировки китайских слов с тремя игровыми режимами
•	Реализация функционального словаря с возможностью поиска и фильтрации
•	Разработка системы добавления и управления пользовательскими словами
•	Создание адаптивного интерфейса с современной цветовой гаммой
## 4. Требования к функциональным характеристикам
## 4.1. Система словаря
•	Просмотр словаря с базовым набором китайских слов (1500+ слов)
•	Поиск слов по иероглифам, пиньиню или переводу
•	Возможность выбора слов для последующей тренировки
•	Добавление слов
•	Удаление пользовательских слов
•	Статистика по количеству найденных и выбранных слов
•	Регистрация пользователя
## 4.2. Система тренировок (3 игровых режима)
Режим 1: Письмо 
•	Отображение китайских иероглифов
•	Поле для ввода пиньиня
•	Проверка правильности ввода с учетом
•	Отображение обратной связи с правильным ответом
•	Пример использования слова
Режим 2: Сопоставление 
•	Отображение пар "иероглифы-перевод" в случайном порядке
•	Drag-and-drop интерфейс для сопоставления
•	Визуальная обратная связь о правильности сопоставления
•	Автоматическая проверка при завершении всех пар
Режим 3: Выбор варианта
•	Отображение слова на Китайском языке
•	3-4 варианта перевода (1 правильный, остальные случайные)
•	Выбор правильного варианта
•	Визуальная подсветка правильного/неправильного ответов
## 4.3. Система управления тренировками
•	Выбор от 1 до 20+ слов для тренировки
•	Случайный выбор слов при отсутствии явного выбора
•	Отслеживание прогресса в реальном времени:
•	Визуальный прогресс-бар
•	Подведение итогов после завершения тренировки
## 5. Требования к надежности
•	Валидация всех входных данных на фронтенде и бэкенде
•	Защита от SQL-инъекций через параметризованные запросы
•	Контроль уникальности идентификаторов пользовательских слов
•	Обработка исключительных ситуаций с логированием
•	Резервное копирование пользовательских данных в localStorage
•	Проверка доступности бэкенда перед отправкой запросов
## 6. Условия эксплуатации
•	Веб-доступ через стандартные браузеры (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
•	Поддержка мобильных устройств с адаптивным дизайном
•	Работа в локальной сети и через интернет
•	Минимальное разрешение экрана: 320×480 (мобильные устройства)
## 7. Требования к составу и параметрам технических средств
Серверная часть:
•	ОС: Linux (Ubuntu/Debian)
•	Python 3.8+
•	Процессор: 1 ядро, 1.0 ГГц
•	ОЗУ: 512 МБ
•	Дисковое пространство: 100 МБ
•	Веб-сервер: Nginx 1.18+
•	База данных: SQLite 3
Клиентская часть:
•	Веб-браузер современной версии
•	Поддержка JavaScript ES6+
•	Интернет-соединение: от 500 Кбит/с
•	Поддержка локального хранилища (localStorage)
 
## 8. Требования к информационной и программной совместимости
Backend:
•	Flask 2.3+
•	Flask-CORS для поддержки межсайтовых запросов
•	Встроенная библиотека JSON для работы с данными
•	Стандартная библиотека Python для работы с файлами
Frontend:
•	Чистый HTML5, CSS3, JavaScript (ES6+)
•	Без использования внешних фреймворков
•	Поддержка современных веб-стандартов
•	Использование Fetch API для HTTP-запросов
Стандарты:
•	REST API архитектура
•	JSON формат данных для API
•	Адаптивный дизайн (responsive design)
•	Progressive Web App принципы
## 9. Требования к данным и хранению
## 9.1. Форматы данных
•	Основной словарь: JSON формат с полями
•	Пользовательские слова: JSON формат
•	Настройки пользователя: JSON в localStorage
## 9.2. Хранение данных
•	Основной словарь: файл dictionary.json
•	Пользовательские слова: файл user_words.json
•	Сессии пользователя: браузерный localStorage
•	Статистика тренировок: браузерный localStorage
